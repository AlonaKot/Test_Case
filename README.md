# Test_Case

Форма для тестирования: 
+ http://itcareer.pythonanywhere.com/ 

Требования к форме:
1. Форма регистрации пользователя
2. Поле Name и Surname:
+ только латиница
+ min 5 символов
+ max 128 символов
+ не регистрозависимые
+ спец символы - только пробел и точка
+ 2 точки подряд ставить нельзя
+ 2 пробела подряд ставить нельзя
+ копипаст можно

3. Поле Password
+ пароль должен булиться (прятаться) при вводе
+ минимум 6 символов 
+ макс 15 символов
+ только латинцица
+ обязательно 1 собака (может содержать больше 1)
+ обязательно 1 заглавная буква
+ обязательно 1 маленькая буква
+ обязательно 1 цифра
+ остальные спец.символы запрещены и пробелы
+ можно копировать - вставить
+ валидация на популярные пароли (123456)

4. Поле E-mail
+ Не проверяется существует или нет
+ Всего 256 символов

+ Формат: до собаки от 5 до 48 символов
+ цифры до @ и до последней точки
+ нельзя пробел, нижнее подчеркивание, нельзя слеш. Можно только точку и тире
+ @ только 1 раз
+ 2 точки подряд ставить нельзя
+ 2 тире подряд ставить нельзя
+ не регистрозависимый
+ можно только цифры, можно только буквы и цифры + буквы
+ Если ввод на кирилице должно біть сообщение "не вал=идно заполнено поле" после нажатия сабмит
 
+ после @ до точки :
+ точки 2 подряд нельзя
+ можно тире

+ от @ до последней точки 
+ мин 2 символа, максимум 32

+ после последней точки
+ мин 2 символа максисум 16
 
4. Обязательные поля (должны быть отмечены звездочками):
+ Name 
+ E-mail
+ Password
+ Если не заполнены все обязательные поля, после нажатия "Submit" должно появиться сообщение "Заполните все обязательные поля"

5. Форма не должна сохранять введенные данные после обновления страницы

6. Между полями можно перемещаться нажатием кнопки ТАВ

7. 
+ реакция UI на неверные данные - выделено красной рамочкой поле и внизу месседж "у вас не верно заполнены поля" (отрабатывает только после нажатия  "Submit"
+ после исправления ошибок и нажатия "Submit" все ошибки должны пропадать
+ если верно заполнены поля должно быть сообщение после "Submit" - Hello, "NAME SURNAME"

ОТРАБОТКА ВАЛИДАЦИОННЫХ СКРИПТОВ:
+ валидация нажатием сабмит
+ при снятии фокуса с поля происходить валидация введенных данных
+ IN TIME - не дает вводить доп символы (УДАЛЯЕТ) джава скрипт

8. Верстка адаптивно-респонсивная
9. кукис не работает

10. кнопка сабмит:
+ текст по центру
+ выход теста принятия данных - синий
